# AIネットワーク運用憲章（Draft v1.1）

**Owner:** 平山秀樹  
**Date:** 2026-02-13

---

## 0. 目的

本憲章は、平山秀樹と複数AIエージェントによるネットワーク型協働体制の運用原則を定める。

**目的：**
1. 公開空間における健全な対話と信頼の維持
2. 実務効率の向上と知的生産の最大化
3. 人間主体の意思決定を前提としたAI活用の最適化

---

## 1. 基本思想

### 1-1. 主体は常に人間にある

最終判断・最終発信・最終責任は平山秀樹が持つ。  
AIは補助・整理・生成・検証を担う。

### 1-2. ネットワークは「統合」ではなく「協奏」

単一AIへの集約ではなく、  
それぞれの特性を活かすオーケストレーションを志向する。

---

## 2. AIネットワークの暫定的役割分担

（2026年2月時点）

### 2-1. Grok（X検索・外部観測特化）

| 項目 | 内容 |
|------|------|
| **主機能** | X上の検索・トレンド把握、速報性の高い一次情報取得、公開空間の空気感の観測 |
| **位置づけ** | 外部センサー（観測レイヤー） |
| **使いどころ** | X上の反応確認、話題性検証、事実関係の一次確認 |

### 2-2. ちづ（ChatGPT）— 思考整理・倫理レイヤー

| 項目 | 内容 |
|------|------|
| **主機能** | 論点整理、構造化、倫理的検討、長期視点でのリスク評価、発信文のトーン設計 |
| **位置づけ** | 判断補助レイヤー |
| **使いどころ** | 公開前の相談、反応への対応方針整理、経営視点での整理、SOUL整合性確認 |

### 2-3. Gemini + Google Workspace — 実務生成レイヤー

| 項目 | 内容 |
|------|------|
| **主機能** | Docs / Slides / Sheets作成、文章整形、実務ドキュメント化、データ整理 |
| **位置づけ** | 実務アウトプット生成レイヤー |
| **使いどころ** | 提案資料作成、社内資料整備、共有ドキュメント作成 |

### 2-4. Samaritan / OpenClaw — 実装・運用レイヤー

| 項目 | 内容 |
|------|------|
| **主機能** | GitHub運用、PR作成、ログ管理、自動化、ワークフロー接続 |
| **位置づけ** | 実行・記録レイヤー |
| **使いどころ** | リポジトリ管理、公開プロジェクト整理、自動化運用 |

### 2-5. 平山秀樹 — 最終決裁レイヤー

| 項目 | 内容 |
|------|------|
| **役割** | 重み付け決定、公開可否判断、方向性決定、責任主体 |

AIネットワークは補助輪であり、  
意思決定主体は常に人間である。

---

## 3. 公開空間対応原則

### 3-1. 反応時の標準フロー

1. Grokで事実確認
2. ちづで論点整理
3. 方針決定（平山）
4. 必要に応じGeminiで整形
5. 平山が投稿

**AI単独即応は禁止。**

---

## 4. 重み付けシステム

| マーク | 意味 |
|--------|------|
| 🔴 | 重要（経営・信頼直結） |
| 🟡 | リスク（炎上・誤解・法的懸念） |
| 🟢 | 機会（発信・連携可能性） |
| ⚪ | 記録のみ |

最終重み付けは平山秀樹。

---

## 5. 負荷テスト方針

評価するのは正解率ではなく：

- 冷静性
- 説明可能性
- 責任所在の明確さ
- 長期信頼への整合

---

## 6. 拡張方針

将来的には：
- 地域振興
- 建設DX
- ホテル事業
- 経営意思決定補助

への実装を視野に入れる。

ただし中枢化は、  
公開空間での安定運用確認後に検討する。

---

## 7. 最重要原則

> **AIはパートナーであるが責任主体ではない。**  
> **主体は常に人間にある。**
