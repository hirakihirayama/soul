## CLAUDE.md との整合解釈

「AIに言われるがまま」を文字通り取ると CLAUDE.md（制御しない、誘導しない、判断は代行せず選択肢を示す）と矛盾する。

解釈：**秀樹さんが消したいのは「選択そのもの」ではなく「選択疲労」**。
→ 補助輪は「バックログ100個から候補3個に絞る」まで。3個から1個を選ぶのは秀樹さん。これなら CLAUDE.md と矛盾せず、脳の選択コストは大きく下がる。

## 既存 PDCA Tracker の問題点（コードを読んで特定）

### 実装上
- **A.** ループ内 `getRange().getValue()` で API 往復が多い → `getValues()` 一括で10倍速以上
- **B.** TaskID列（H列）が `setColumnWidth(8, 1)` で1px。誤クリック可能 → `hideColumns(8)`
- **C.** Plan列に書きかけのまま1時間タイマーが走ると、未確定の文字列が Tasks に登録される事故。「確定」フラグなし
- **D.** 繰越時に Google Tasks 側の `due` が更新されない（前日のまま期限切れ表示）
- **E.** 「繰越」ステータスがプルダウンと自動更新で二重定義
- **F.** 7日超のブランク（連休・出張・入院）で未完了が消える（最大7日遡り）
- **G.** タブが無限増殖（1年で365、3年で1100超）。アーカイブなし

### 運用上（PDCA として機能しているか）
- **H.** Check / Action 列がほぼ書かれない構造（書く動線がない）
- **I.** Action が翌日の Plan に繋がらない（連鎖なし。Action 欄が孤立）
- **J.** 完了タスクの成果物リンクが日付タブに散逸し後から探せない
- **K.** 「予定」と「タスク」が同列でステータス管理されノイズになる

### 「補助輪の思想」観点でのリスク
- 自動繰越は「明日に回せばいい」を既定にし、**捨てる判断を削る**
- 繰越時の前日シート書き換えに取り消し手段なし
- Chat 通知文「明朝自動で繰り越されます」は安心感を与えすぎてタスク増殖を促す

## 再設計方針

### 必要なタスク属性（現状にない）

| 属性 | 値 | 用途 |
|------|---|------|
| 目的タグ | K/N/B/A（複数可、なしも可） | 第二領域判定 |
| 重要度 | 高/中/低 | 象限判定 |
| 緊急度 | 高/中/低 | 象限判定（期日から自動算出も可） |
| 領域 | I/II/III/IV | 重要×緊急から自動算出 |
| 期日 | 日付 | 緊急度算出 |
| 想定時間 | 15分/30分/1h/半日/1日 | 1日の枠に収まるか判定 |
| 投入日 | バックログに入れた日 | 「1週間ルール」判定 |
| 委譲先 | 人名 or 空欄 | 第三領域処理 |

### ルーチン1：朝の「今日3つ」提示
- AI がバックログから候補3つを提示
- 必ず1つは K/N/B/A タグ持ち（第二領域を毎日1枠確保）
- 期日近いタグなし1つ（第一領域処理）
- 残り1つは状況依存（午前は重い思考、午後は軽い処理など）
- 秀樹さんは「OK / 別の日 / 捨てる」を1クリック

### ルーチン2：週次の消し込み（金曜17時）
- 「**タグなし & 投入から7日経過**」を一覧表示
- 各タスクに3択：**やる日を決める / 委譲する / 捨てる**
- 「保留」は選ばせない（残すなら必ず行動を決める）
- これが「優先度低いものを1週間で消し込む」の自動化

### ルーチン3：第二領域の時間ブロック
- カレンダーに第二領域専用の時間枠を毎日自動投入（例：午前10〜11時）
- そこに K/N/B/A タグタスクを充てる

### KPI のリアリズム
4目的を日次で全部回すのは無理。実態の頻度：
- **K**：月に数回（書類・面談） → **週1**触る
- **N**：構造的な動き → **週1**触る
- **B**：毎日（運動・睡眠・食事）
- **A**：ほぼ毎日（社内推進）

KPI は「日に1個ずつ」ではなく **「週に最低1回は K/N/B/A 全部に触る」**。週次ダッシュボードで「今週 K に何分／何回触ったか」を可視化し、放置目的を見える化。

## 実装の選択肢（次セッションで決める）

1. **現 PDCA Tracker を拡張**：既存資産活用、ただし日次ベースに優先順位ロジックを後付け
2. **別シート「マスター・バックログ」を新設し PDCA Tracker と分離**（**推奨**）：実行ログと優先順位付けは脳の使い方が違うので分けた方が思考が混ざらない
3. **GWS を諦めて Notion / Sunsama / Motion 等**：第二領域管理ネイティブだが秀樹さんが GWS で完結したい意思がある場合は却下

### 案2 の構造案
- **マスター・バックログシート**：全タスク。属性列（目的タグ、重要度、緊急度、期日、想定時間、投入日、委譲先、ステータス）
- **日次タブ**（既存 PDCA Tracker）：マスターから「今日の3つ」を引いてくる実行ログ。Check / Action 記入欄
- **週次サマリータブ**：金曜生成。K/N/B/A 別の今週投入時間、未着手バックログの消し込み画面
